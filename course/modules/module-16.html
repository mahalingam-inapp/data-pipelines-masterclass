Module 16: From Pipeline to Analytics

How Do Reports Connect to the Pipeline?

Curated or Gold tables are the source of truth for reports and dashboards. Refresh can be scheduled (e.g. daily). A semantic layer adds business names and definitions. Reports can use import (data copied into the tool) or DirectQuery (live query); incremental refresh and row-level security (RLS) are common requirements.

Key insight: Trace a filter click back to the pipeline: which table and column? Lineage (report → dataset → table → run) supports trust and troubleshooting. Refresh frequency follows pipeline schedule and tool choice.

Connecting reports to the pipeline

Trace a filter click in a report back to the pipeline: which table and column? Lineage (report → dataset → table → pipeline run) supports trust and troubleshooting. Refresh frequency (real-time vs. daily vs. weekly) is a product of pipeline schedule and tool choice.

Practical use cases

  • Daily refresh of a Power BI dataset from a Gold table.
  • Tableau extract on a mart with weekly refresh.

Pulling Gold data for reports: Python and R

Process: Reports consume Gold or curated tables. You can pull the same data in Python or R to build custom analyses, validate numbers, or export a snapshot for a BI tool. Components: connection (DB or file path), query or read call, optional filters and date range. Outcome: a dataframe or table ready for charts or for feeding Power BI/Tableau (e.g. CSV/Parquet import).

Python: read Gold from warehouse or Parquet for a report
import pandas as pd
# From Parquet (e.g. lake or mart export)
df = pd.read_parquet("gold/daily_sales_by_region/")
df = df[df["sale_date"] >= "2025-01-01"]
# Or from warehouse (e.g. snowflake.connector, sqlalchemy)
# df = pd.read_sql("SELECT * FROM gold.daily_sales WHERE sale_date >= '2025-01-01'", con)
# Export for Power BI import or Tableau extract
df.to_csv("exports/sales_for_report.csv", index=False)
R: read Gold and run a quick sanity check before reporting
library(arrow)
library(dplyr)
gold <- read_parquet("gold/daily_sales_by_region/") %>% filter(sale_date >= as.Date("2025-01-01"))
# Sanity: row count and revenue range
nrow(gold)
summary(gold$revenue)
# Export for BI tool if needed
readr::write_csv(gold, "exports/sales_for_report.csv")

Module Quiz

Q1: What is the typical source of truth for a report?

  • A) Bronze table
  • B) Curated or Gold table
Correct: B. Curated/Gold are the business-ready layers; Bronze is raw.

Q2: What is a semantic layer?

  • A) Raw storage
  • B) Business names and definitions (metadata) so report consumers see consistent terms
Correct: B. Semantic layer adds business metadata for consistency and self-serve.

Q3: What is row-level security (RLS) used for?

  • A) To make reports faster
  • B) To restrict which rows a user sees based on role or attribute (e.g. region)
Correct: B. RLS enforces access at the row level in the dataset or report.

Q4: Why trace a report filter back to the pipeline?

  • A) To slow down development
  • B) For trust, debugging, and lineage—to know which table and column feed the report
Correct: B. Lineage supports governance and troubleshooting.

Q5: What drives the choice of refresh frequency for a dashboard?

  • A) Only the tool
  • B) Business need (how fresh data must be), pipeline schedule, and tool capability (import vs. DirectQuery)
Correct: B. Refresh is a product of pipeline schedule, latency requirement, and BI tool mode.
