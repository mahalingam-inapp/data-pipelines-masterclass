Module 21: Simulation & Holistic View

What This Module Covers

Simulation lets you practice failure and recovery in a safe setting (e.g. missing file, schema change, duplicate key). The holistic view ties the course together: business question → data need → pipeline design → storage → report → action.

Key insight: Breaking the pipeline on purpose (missing file, schema change, duplicate key) and fixing it in a time box builds operational confidence. The "question → pipeline → report" roundtable connects everything to real business questions.

Simulation: Failure and recovery

Scenarios to practice:

  • Source file missing → pipeline fails → fix (retry, fallback, alert) and re-run.
  • Schema change (new required column) → pipeline breaks → update Silver and Gold and deploy.
  • Duplicate key in source → duplicate rows in Gold → add dedup and backfill.

Instructor or script “breaks” the pipeline; teams diagnose and fix in a time box; debrief on steps. Rotate: one person breaks, another fixes.

Holistic view: Business question → answer

1. Start with a business question (e.g. “Why did churn increase in Q3?”).
2. Map to data need: which events, billing, support; which layers; which mart; which report.
3. Discuss what’s missing and what’s next (e.g. real-time, ML).

Roundtable: each participant brings one real business question; group picks two and walks through “question → pipeline → report” on a whiteboard. One-pager: data request template (question, needed data, suggested pipeline change, expected output).

Key Takeaways

  • Simulation: missing file, schema change, duplicate key; diagnose and fix in a time box; debrief on steps.
  • Holistic: business question to data need to pipeline to storage to report; discuss what is missing and what is next.
  • Data request template: question, needed data, pipeline change, expected output.

Real-World Applications

  • Ops drill: Rotate who breaks and who fixes; document runbook updates from each scenario.
Relationship to other modules: Module 20 was the capstone build; this one stresses troubleshooting and end-to-end thinking. Module 22 is the roadmap and next steps.
